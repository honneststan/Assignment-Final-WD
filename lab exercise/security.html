1.What is the Hypertext Transfer Protocol (HTTP)?
Hypertext Transfer Protocol (HTTP) is an application-layer protocol for transmitting hypermedia documents, such as HTML. It was designed for communication between web browsers and web servers.
2.Why is HTTP called a stateless protocol?
HTTP is called a stateless protocol as the server does not keep any data/state between 2 requests.
3.Describe the three phases of establishing a session using client-server protocols such as HTTP.
In client-server protocols, like HTTP, sessions consist of three phases:
  1. The client establishes a TCP connection (or the appropriate connection if the transport layer is not TCP).
  2. The client sends its request, and waits for the answer.
  3. The server processes the request, sending back its answer, providing a status code and appropriate data.
4.Describe the parameters on a get and post request.
On a GET request, the parameters are stored in the form of name=value and these parameters appear in the URL. Multiple parameters can appear after the ? sign in the URL which are seperated by & sign.
On a POST request, the parameters do not appear on the URL. POST carries request parameter in message body which make it secure.
5.What are the HTTP response status code and What do the HTTP status codes, 200, and 404 indicate?
HTTP response status codes indicate whether a specific HTTP request has been successfully completed. HTTP status Codes:
  • 200 means request has succeeded.
  • 404 means the server cannot find the requested resource.
6.What are HTTP cookies? Give an example.
An HTTP cookie (web cookie, browser cookie) is a small piece of data that a server sends to the user's web browser.
Cookie: yummy_cookie=choco; tasty_cookie=strawberry
a web server might send a cookie containing the username that was last used to log into a website, so that it may be filled in automatically the next time the user logs in.
7.Why are cookies a security issue?
Cookies are often used in web application to identify a user and their authenticated session, so stealing a cookie can lead to hijacking the authenticated user's session.
